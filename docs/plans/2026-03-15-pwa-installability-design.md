# PWA Installability Design

**Date:** 2026-03-15
**Author:** Claude (via brainstorming skill)
**Status:** Approved

## Overview

Make the VoiceScribe Progressive Web App (PWA) installable on both mobile and desktop platforms by generating PNG icons from existing SVG assets and updating the manifest configuration.

## Problem Statement

The app has:
- `next-pwa` configured
- Valid `manifest.json` with metadata
- SVG-based app icons

However, PWA installability requires:
- PNG format icons (many browsers don't support SVG for install prompts)
- Multiple icon sizes for different platforms
- The manifest shortcuts reference non-existent PNG files

## Solution: Local Icon Generation with Sharp

### Approach

Add a build script using the `sharp` library to convert existing SVG icons to PNG format in multiple sizes required for PWA installability across all platforms.

### Files to Create/Modify

#### 1. New: `scripts/generate-icons.js`

A Node.js script that:
- Reads the existing `icon-192.svg` and `icon-512.svg` from `web/public/`
- Converts them to PNG format
- Generates multiple sizes for platform compatibility
- Saves to `web/public/icons/` directory

#### 2. Modify: `web/package.json`

Add new entries:
- `devDependencies`: Add `sharp`
- `scripts`: Add `"generate-icons": "node scripts/generate-icons.js"`

#### 3. Update: `web/public/manifest.json`

Update icon references:
- Change from SVG to PNG file paths
- Fix shortcuts icons to reference valid PNG files

### Icon Sizes to Generate

| Size | Platform Purpose |
|------|------------------|
| 72x72 | Android low-density |
| 96x96 | Android medium-density |
| 128x128 | Chrome Web Store icon |
| 144x144 | Android high-density |
| 192x192 | PWA standard, Android adaptive icon |
| 384x384 | PWA 2x scale factor |
| 512x512 | PWA large size, iOS splash screen |

### Implementation Flow

```
1. Developer runs: npm run generate-icons
2. Script reads icon-192.svg and icon-512.svg
3. Sharp converts to PNG at multiple sizes
4. PNGs saved to web/public/icons/
5. Manifest updated to reference PNG icons
6. PWA becomes installable on all platforms
```

### Error Handling

- Script checks if `sharp` is installed before running
- Validates source SVG files exist
- Reports success/failure for each icon size
- Creates target directory if missing

## Testing Checklist

After implementation, verify:

- [ ] Run `npm run generate-icons` successfully
- [ ] All PNG icon files are created in `web/public/icons/`
- [ ] Chrome DevTools Lighthouse PWA audit passes
- [ ] "Install" prompt appears on Chrome desktop
- [ ] "Add to Home Screen" prompt appears on mobile browsers
- [ ] App launches correctly after installation
- [ ] Icons display properly on home screen/app drawer

This design will be preserved in the PNG conversions.
